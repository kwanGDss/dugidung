# 두기둥 v0.2 — 결과 페이지 풍성화 + 랜딩/로딩 폴리싱

- 생성일: 2026-04-14
- 상태: 승인 대기
- 선행: v0.1.0 (`docs/superpowers/specs/2026-04-14-dugidung-design.md` + `dugidung-v0.1.0` 태그)

## 한 줄 요약

결과 페이지를 6가지 차원(사주 깊이 / 관계 조언 / 바이럴 훅 / 시각 / 투명성 / 분량)에서 전부 풍성화한다. LLM 한 번 호출로 편지 + 아키타입 + 계절 3카드를 동시 생성하고, 두 사람의 연/월/일주 오행 분포를 계산해 노출한다. 랜딩 배경 이미지를 추가하고, 제출 후 로딩 오버레이를 넣어 몰입감을 올린다.

## 목적

- 한 번 보고 끝나는 결과 페이지가 아닌, **공유·재탐색 가치**를 만드는 페이지로
- 사주보자 팬에게는 깊이를, 처음 보는 사람에게는 훅을
- 카톡으로 링크 받은 사람이 **자기도 해보고 싶게**

## 비목표

- 사주 전문가용 상세 해석(십신·조후용신 등)
- 계정·히스토리
- AI 챗봇 follow-up (별도 스펙)
- 시주(時柱) 입력

## 1. 데이터 모델 변화

### 1-1. `Letter` 타입 확장 (`lib/types.ts`)

```ts
export interface Archetype {
  name: string;         // 4~10자, 자연물 비유 기반 유형명
  description: string;  // 30~60자 한 줄 설명
}

export interface Seasons {
  strength: string;     // 30~80자, "이 관계가 잘 굴러가는 이유"
  tension: string;      // 30~80자, "어디서 미끄러지는지"
  advice: string;       // 30~80자, "일상에서 바로 쓸 수 있는 행동 한 가지"
}

export interface Letter {
  title: string;
  body: string;
  pullQuote: string;
  archetype: Archetype;              // NEW
  seasons: Seasons;                  // NEW
  promptVersion: string;
  source: "llm" | "fallback";
}
```

### 1-2. `Pillars` 확장 (`lib/types.ts`)

오행 분포 계산용. 기존 `element`(일간 오행) 는 유지하되 전체 오행 배열 추가.

```ts
export interface Pillars {
  year: string;
  month: string;
  day: string;
  zodiac: Zodiac;
  element: Element;            // 일간 오행 (기존)
  elements: Element[];         // NEW — 연/월/일주 천간·지지 6개 (시주 미포함)
}
```

### 1-3. 오행 분포 모듈 (`lib/ohaeng-distribution.ts`, 신규)

```ts
import type { Element, Pillars } from "@/lib/types";

export interface Distribution {
  목: number;
  화: number;
  토: number;
  금: number;
  수: number;
}

export function combineDistribution(a: Pillars, b: Pillars): Distribution {
  const out: Distribution = { 목: 0, 화: 0, 토: 0, 금: 0, 수: 0 };
  for (const el of [...a.elements, ...b.elements]) out[el]++;
  return out;
}
```

### 1-4. `saju.ts` 확장

지지(地支) → 오행 매핑 추가:

```ts
const BRANCH_TO_ELEMENT: Record<string, Element> = {
  자: "수", 축: "토", 인: "목", 묘: "목",
  진: "토", 사: "화", 오: "화", 미: "토",
  신: "금", 유: "금", 술: "토", 해: "수",
};
```

`birthToPillars` 는 연/월/일주 각각의 천간·지지를 돌며 `elements: Element[]` 를 생성해서 반환.

### 1-5. CompatRecord 버전 bump

```ts
export interface CompatRecord {
  version: 2;   // was 1
  ...
}
```

`defaultStore().get<CompatRecord>` 직후 `record.version !== 2` 체크 → `null` 반환(miss). v1 캐시는 자동 폐기되고 새 구조로 재생성됨. 캐시 키 prefix 는 그대로 `compat:<hash>`.

### 1-6. 불변식

- 같은 입력 → 같은 해시 → 같은 결과 (아키타입 포함 전부 캐시)
- 오행 분포는 순수 계산 (LLM 무관, 결정론적)
- `source: "fallback"` 일 때도 archetype/seasons 필드는 반드시 채워짐

## 2. LLM 프롬프트 변화

### 2-1. 단일 호출

편지 + 아키타입 + 계절 3카드 모두 한 번의 LLM 호출로 생성. 출력 토큰 ~400 → ~650 증가. 비용 영향 미미 (gpt-4o-mini 쌍당 ~$0.0005).

### 2-2. `PROMPT_VERSION = "2.0"`

`lib/prompts.ts`. KV 레코드의 `letter.promptVersion` 은 "2.0" 으로 기록되고, 추후 프롬프트 개선 시 "2.1" 등으로 올림. 기존 레코드는 CompatRecord.version bump 로 자동 폐기되므로 문제 없음.

### 2-3. system prompt 추가 블록

```
너의 출력은 편지 한 편, 그 커플의 유형 이름, 그리고 관계의 세 면(강점/긴장/조언)을 포함한다.

아키타입:
- 자연물 비유 기반 유형명 — 4~10자, 새로 지어낸 이름이어야 하며
- "천생연분", "불꽃커플", "운명의 짝" 같은 밈 클리셰 금지
- description 은 30~60자, 이 유형이 어떤 관계인지 한 줄로 풀어서

계절 3 카드 (각각 30~80자, 1~2문장):
- 강점: 이 관계가 잘 굴러가는 이유. 구체적으로. 관념적 찬사 금지.
- 긴장: 어디서 미끄러지는지. 비난이 아닌 관찰 톤.
- 조언: 일상에서 바로 쓸 수 있는 한 가지 행동. 추상 금지.
```

십이지 65 × 0.25 = 16.25
